通过协作和标准推动PHP向前发展.

欢迎来到PHP框架互操作组！我们是一组已建立的PHP项目，其目标是讨论我们的项目之间的共同点，并找到我们可以更好地一起工作的方法。

查看推荐规范(PSRs) 常见问题

  • 自动加载(Autoloading)

    自动加载器去除通过映射命名空间到文件系统路径来包含文件的复杂性.

    • PSR-4: 改进的自动加载（Improved Autoloading）
    <?php

use Vendor\Package\ClassName;

$object = new ClassName();

  • 接口（Interfaces）

    接口通过遵循预期的契约简化了项目之间的代码共享。

    • PSR-3: 日志接口（Logger Interface）
    • PSR-6: 缓存接口（Caching Interface）
    • PSR-11: 服务容器接口（Container Interface）
    • PSR-13: 超媒体链接（Hypermedia Links）
    • PSR-14: 事件调度程序（Event Dispatcher）
    • PSR-16: 简单缓存（Simple Cache）
    • PSR-20: 时钟（Clock）
    <?php

namespace Psr\Log;

/**
 * Describes a logger instance
 */
interface LoggerInterface
{

  • HTTP

    可互操作的标准和接口, 在客户端和服务器端采用agnostic approach来处理 HTTP 请求和响应.

    • PSR-7: HTTP消息接口（HTTP Message Interfaces）
    • PSR-15: HTTP处理程序（HTTP Handlers）
    • PSR-17: HTTP工厂（HTTP Factories）
    • PSR-18: HTTP客户端（HTTP Client）
    <?php

namespace Psr\Http\Message;

/**
 * Representation of an outgoing, client-side request.
 */
interface RequestInterface extends MessageInterface
{

  • 编码风格（Coding Styles）

    标准化格式减少了阅读其他作者的代码时的认知摩擦.

    • PER Coding Style
    • PSR-1: 基础编码标准（Basic Coding Standard）
    • PSR-12: 扩展编码风格指南（Extended Coding Style Guide）
    <?php

namespace Vendor\Package;

class ClassName
{
    public function fooBarBaz($arg1, &$arg2, $arg3 = [])
    {
        // method body
    }
}
